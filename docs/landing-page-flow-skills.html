Installable workflow package

Bring governed SDD into the daily dev loop.

flow-skills is an installable npm package for teams that want structured planning, execution, review and validation without turning developer work into prompt chaos. It ships as a real package, with CLI lifecycle, reusable skills and an explicit artifact chain.

Quick start
npm i -D @padroes-dev-ia/flow-skills
npx flow-skills init --project . --tools codex,claude --profile strict
npx flow-skills status --project . --feature my-feature
npx flow-skills verify --project . --feature my-feature

What this solves for developers

Most teams do not fail because they lack prompts. They fail because planning, implementation, review and validation drift apart. flow-skills closes that gap.

Clear workflow state

Developers can inspect where a feature stands with status, ask for the next action with next, and audit workflow alignment with verify.

Less context waste

Planning and execution are split into distinct skills and isolated worker contexts, reducing prompt bloat and accidental contamination between phases.

Installable, not ad hoc

The solution is distributed as a real npm package with init, update, managed skill sync and per-project config.

Review before closure

It creates room for structured review, fix-review loops and evidence-oriented verification before anyone claims a feature is done.

Validation planning built in

Teams can generate preliminary validation drafts after task planning and promote them to final validation plans after clean review.

Docs that do not rot

Generic repository docs can be drafted with flow-doc-workshop so README, runbook, onboarding and migration guidance follow a real process too.

Why teams adopt it

flow-skills is useful when the team wants stronger engineering discipline without building a heavyweight internal platform first.

It helps when you need

  • predictable planning before code starts
  • artifact-driven execution instead of vague prompt chains
  • clear workflow conventions from the first project setup
  • reusable workflow conventions across repositories

It helps developers by reducing

  • rework caused by missing PRD or TechSpec decisions
  • review loops with no issue tracking discipline
  • manual install drift across agent environments
  • documentation written too late or from memory
Not just prompts. A governed developer workflow product.

Install once, configure per project, and give the team a repeatable path from idea to validated delivery.
