Spec-Driven Development for shipping teams

Turn feature work into a governed delivery flow.

flow-sdd is an installable npm package that brings a structured PRD -> TechSpec -> Tasks -> Review -> Validation workflow into real repositories. It works for single-project delivery and for multi-project features that span APIs, workers, frontends, databases, and supporting services.

What it helps developers do

  • Plan features before implementation pressure collapses the scope.
  • Keep business intent, technical design, tasks, review, and validation aligned.
  • Stop guessing the next step during long feature work.
  • Coordinate work that spans more than one repo or service.
  • Install the same workflow into more than one LLM client without rewriting prompts.

What it avoids

  • untracked feature drift
  • tasks detached from the spec
  • review without artifact context
  • multi-service rollout with no release notes

Artifact chain

Designed around SDD, not ad hoc prompts

CLI visibility

Know what is blocked, what is next, and what is missing

  • status inspects artifacts, phases, grouped project progress, and blockers.
  • next resolves the next executable task and avoids dependency-blocked work.
  • verify audits completeness, correctness, and coherence.
  • impact, projects, and contracts support multi-project coordination.

Multi-LLM

Use one workflow across clients

  • Codex via .agents/skills/
  • Claude via .claude/skills/
  • Gemini, Cursor, or custom tools via configurable toolTargets

Release-aware

Better for real production change

  • _impact-map.md for affected projects
  • _contracts.md for service boundaries
  • _release-plan.md for deployment order
  • _rollback-plan.md for mitigation and recovery
